## Завдання 1. Мережева діагностика

### 1.1. Виведення IP-адрес та інтерфейсів

Виконана команда:

```bash
ip a
```

Короткий коментар:

Команда `ip a` показує всі мережеві інтерфейси системи та їхні IP-адреси.  
Локальна IP-адреса мого інтерфейсу: **вписати свою IP-адресу**, наприклад `192.168.1.25`.

---

### 1.2. Перевірка доступності публічного вузла

Виконана команда:

```bash
ping 8.8.8.8
```

Щоб зупинити виконання команди, потрібно натиснути:

```bash
Ctrl + C
```

Короткий коментар:

Команда `ping 8.8.8.8` перевіряє доступність публічного DNS-сервера Google.  
Якщо є відповіді від `8.8.8.8`, то доступ до інтернету працює.

---

### 1.3. Перевірка відкритих listening-портів

Виконана команда:

```bash
ss -tulpn
```

Якщо потрібні права адміністратора:

```bash
sudo ss -tulpn
```

Короткий коментар:

Команда `ss -tulpn` показує TCP/UDP-порти, які слухають сервіси в системі.  
Приклад сервісу, який слухає порт: **вписати зі свого виводу**, наприклад `sshd` на порту `22`.

---

## Завдання 2. SSH-доступ з ключами та config

### 2.1. Генерація SSH-ключа

Виконана команда:

```bash
ssh-keygen
```

Рекомендований варіант з явно вказаним типом ключа:

```bash
ssh-keygen -t ed25519
```

Короткий коментар:

Команда створює пару SSH-ключів: приватний і публічний.  
Зазвичай ключі зберігаються у файлах `~/.ssh/id_ed25519` та `~/.ssh/id_ed25519.pub`.

---

### 2.2. Копіювання ключа на сервер

Виконана команда:

```bash
ssh-copy-id USER@SERVER_IP
```

Приклад:

```bash
ssh-copy-id student@192.168.1.100
```

Короткий коментар:

Команда копіює публічний SSH-ключ на сервер у файл `~/.ssh/authorized_keys`.  
Після цього можна підключатися до сервера без введення пароля, якщо все налаштовано правильно.

---

### 2.3. Створення або оновлення файлу ~/.ssh/config

Виконана команда:

```bash
nano ~/.ssh/config
```

Приклад вмісту файлу:

```sshconfig
Host myserver
    HostName SERVER_IP
    User USER
    IdentityFile ~/.ssh/id_ed25519
```

Приклад з реальними значеннями:

```sshconfig
Host myserver
    HostName 192.168.1.100
    User student
    IdentityFile ~/.ssh/id_ed25519
```

Для захисту config-файлу можна виконати:

```bash
chmod 600 ~/.ssh/config
```

Короткий коментар:

Файл `~/.ssh/config` дозволяє створити коротке ім’я для підключення до сервера.  
Замість повної команди з IP-адресою можна використовувати короткий запис `ssh myserver`.

---

### 2.4. Підключення до сервера короткою командою

Виконана команда:

```bash
ssh myserver
```

Короткий коментар:

Підключення виконувалося через Host-запис `myserver` з файлу `~/.ssh/config`.  
Пароль під час підключення: **не запитувався / запитувався**.

---

### 2.5. Дані для звіту

Ім’я Host у config:

```text
myserver
```

Чи працює підключення без пароля:

```text
Так, підключення працює без пароля.
```

Або, якщо пароль все ще запитується:

```text
Ні, пароль запитується. Потрібно перевірити ssh-copy-id, права на ~/.ssh та правильність IdentityFile.
```

## Завдання 3. Копіювання файлів між машинами

### 3.1. Створення локального тестового файлу

Виконана команда:

```bash
echo "test" > test.txt
```

Перевірка файлу:

```bash
cat test.txt
```

Короткий коментар:

Було створено локальний файл `test.txt` з текстом `test`.  
Команда `cat test.txt` підтвердила, що файл містить потрібні дані.

---

### 3.2. Передача файлу на сервер через scp

Виконана команда:

```bash
scp test.txt myserver:~/
```

Або повний варіант без SSH config:

```bash
scp test.txt USER@SERVER_IP:~/
```

Короткий коментар:

Команда `scp` скопіювала файл `test.txt` з локальної машини на сервер у домашній каталог користувача.  
Шлях до файлу на сервері: `~/test.txt`.

---

### 3.3. Створення директорії для синхронізації на сервері

Виконана команда:

```bash
ssh myserver "mkdir -p ~/sync_folder"
```

Короткий коментар:

На сервері було створено директорію `~/sync_folder`.  
Вона буде використовуватися для синхронізації файлів через `rsync`.

---

### 3.4. Створення локальної папки для синхронізації

Виконані команди:

```bash
mkdir -p local_sync
echo "file for rsync" > local_sync/file1.txt
echo "second file" > local_sync/file2.txt
```

Короткий коментар:

Було створено локальну директорію `local_sync` з двома тестовими файлами.  
Ці файли будуть передані на сервер за допомогою `rsync`.

---

### 3.5. Синхронізація локальної папки з сервером через rsync

Виконана команда:

```bash
rsync -av local_sync/ myserver:~/sync_folder/
```

Або повний варіант без SSH config:

```bash
rsync -av local_sync/ USER@SERVER_IP:~/sync_folder/
```

Короткий коментар:

Команда `rsync` синхронізувала вміст локальної папки `local_sync` з директорією `~/sync_folder` на сервері.  
Опція `-a` зберігає структуру та атрибути файлів, а `-v` показує детальний вивід.

---

### 3.6. Перевірка файлів через sftp

Підключення через sftp:

```bash
sftp myserver
```

Команди всередині sftp:

```bash
ls
cd sync_folder
ls
bye
```

Короткий коментар:

Через `sftp` було перевірено, що файли присутні на сервері.  
Для перевірки використовувались команди `ls` та `cd sync_folder`.

---

### 3.7. Дані для звіту

Шлях до файлів на сервері:

```text
~/test.txt
~/sync_folder/file1.txt
~/sync_folder/file2.txt
```

Команда для перевірки:

```bash
sftp myserver
```

Або через SSH:

```bash
ssh myserver "ls -la ~/sync_folder"
```
